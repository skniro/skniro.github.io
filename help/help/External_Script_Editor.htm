External Script Editor

 

Visual Novel Maker provides an integrated Script Editor based on the popular ACE Code Editor. However, if you prefer another code editor, VN Maker allows you to use any editor you want for your own custom scripts.

 

Prepare your Project

 

To use an external script editor, navigate to your project folder and create a new folder called "scripts". Now you can just put all your custom scripts in there as .js files and VN Maker will automatically recognize those scripts and add them to your game. You can add a number suffix to a script to control the include order, for example:

 

Fruit-0.js

Apple-1.js

Banana-2.js

 

That is useful in case of inheritance, where it is important that the base-class script is included before the derived classes are included.

 

Pick your Programming Language

 

VN Maker expects your scripts written in JavaScript programming language. If JavaScript is not your favorite language, you can pick any language you want as long as that language can compile to JavaScript such as CoffeeScript, TypeScript, Ruby, etc.

 

Edit Default Scripts

 

You cannot edit the default scripts in an external script editor since they are and need to be managed by VN Maker's DMS (Document Management System). But that isn't a problem at all in regular cases because editing the default scripts directly is never a good idea because your modification get lost if an update/migration happens.

 

If you want to modify a default script, you create copy of it and overwrite the methods and behaviors you want so that your modifications are not overwritten by migration. Then you can also use an external script editor, just copy a default script into the scripts-folder, do you modifications on it and that's it.

 

However, if you want to export all default scripts as external files, you can use the Game Script Exporter script agent via Tools > Script Agents > Default > Game Script Exporter. That exporter allows you to export all scripts as .js or .coffee files. If you check the "Export As JavaScript" check box, only the compiled JavaScript or regular JavaScript will be exported but not the CoffeeScript code.
