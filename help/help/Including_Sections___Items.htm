Including Sections & Items

 

To minimize redundant data, Visual Novel Maker offers advanced include mechanisms.


Including Sections

It is possible to include sections from an already existing view into another view and even overwrite some properties if necessary. To include sections from an already existing view, the following section-definition can be used:

 

{ "include": "<uid>" }

 

That section-definition will include all sections from the specified view. But it is also possible to only include specific sections:

 

{ "include": "<uid>", "sections": ["mySection1", "otherSection"] }

 

The above example includes only the sections with the identifier "mySection1" and "otherSection". All other sections are ignored.  

 

Sometimes it is also necessary to change some properties of the included sections like some labels or section titles. That is possible using the following include-definition:

 

{ "include": "<uid>", "sections": ["mySection1", { "identifier": "otherSection", "overwrite": { "name": "New Title" } } ] }

 

The above example overwrites the title of "otherSection" with "New Title". The overwrite-process just merges the overwrite-object with the section-properties.

 

Including Items

It is possible to include items of a section from an already existing view into another view's section and even overwrite some properties if necessary. To include items, the following item-definition can be used:

 

{ "include": "<uid>", "section": "mySection1" }

 

The above example includes all items of section "mySection1" from the specified view. But it is also possible to only include specific items:

 

{ "include": "<uid>", "section": "mySection1", "items": ["item1", "item2"] }

 

The above example includes only the items with the identifier "item1" and "item2". All other items are ignored.  Sometimes it is also necessary to change some properties of the included items like some labels. That is possible using the following include-definition:

 

{ "include": "<uid>", "section": "mySection1", "items": ["item1", { "identifier": "label", "overwrite": { "text": "New Text" } }] }

 

The above example overwrites the text-property of "label" item with "New Text". The overwrite-process just merges the overwrite-object with the item-properties.
