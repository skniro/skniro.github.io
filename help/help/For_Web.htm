For Web

 

Testing & Deployment for Web platform is very easy. This topic will guide you through the necessary steps.

 

Testing on Web platform

 

Just select Build For > Web from The Menubar for a quick test-build. After that, just navigate to your project-folder or select Game > Open Game Folder from The Menubar. There you will find a build folder with some files and an index.html file inside. You can just open that file with Firefox to play your game.

 

Alternativly you can setup a local web server for more realistic testing or just upload the build-folder to your web-server.

 

Audio & Video Assets

 

At time of writing, one half of the Web-Browsers only supports .m4a/.mp4/.mp3/.aac/.wav for Audio and .mp4 for Video while the other half only supports .ogg/.wav for Audio and .webm for Video. If you plan to support all major browsers you have to make sure that your game contains all Audio and Video files in at least two formats. Only exception are .wav files used for simple sound effects.

 

VN Maker allows you to import the same file multiple times with different formats and then automatically loads the correct file at runtime.

 

NOTE: Video-Playback might not work correctly in some mobile web-browsers because of security restrictions.

 

Web Start Logo

On some platforms, especially on mobiles, if you load your game with a mobile webbrowser, your game will show up a web-start screen which requires the player to tap on it one time to start the actually game. That is necessary because of the strict security of mobile browsers.

 

 

You can find and exchange the web-start logo in Resource Manager if you navigate to Graphics/Pictures folder and look for web-start-screen image file.

 

Deploy for Web platform

 

To deploy your game for Web, you can either choose Build For > Web from The Menubar for a quick test-build or use the Create Game Package wizard to create a ready-to-ship web package. You can now upload that web-package to your web-server and share it with the world!

 

If you want to take more control over how your game will be packaged, take a look at Build & Run documentation.
