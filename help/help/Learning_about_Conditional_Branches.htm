Example #1: A Simple Counter

For this tutorial, we'll make a fairly simple example of how to use the Condition command. First, let's take a look into this flowchart:

 

 

As to how this reads, the flowchart goes with the logic, if Counter is equal to 6, the process will end. Else it will add +1 to the Counter and check again. If we convert it to Visual Novel Maker Format, it will look like this:

 

 

So here's a quick breakdown:

 

  • I used a Local Number Variable for this particular example. This is because the variable will not be used anywhere else in the game. It is only unique to this Scene.
  • We added Message Box Visibility command and set the Visible to Yes. This is so we can use a Show Message command to show the value of the variable we're using with the help of a text code, {LN:1}. Which you can see under the Else command.
  • We created a Label called 'start' since we want the condition to return here after adding a value to the variable.
  • Comment is a command that is ignored by the engine and only serves as reminders to the developer. Don't be afraid to use them!

 

If you use Play Test, you will see the number ticking up to five. We made a simple counter!

 

Example #2: Creating a Treasure Chest

For our next example, we are going to make a small text prompt that asks if we want to open a treasure chest. It will give a different response if you already opened the chest. Here's a quick flowchart that explains how this will work:

 

It looks a bit more complicated but it's really the same as our first example! The only difference is that we now have a 'branching point' because we're using choices to deliver a different outcome. If we convert it to Visual Novel Maker Format, it will look like this:


 

So here's a quick breakdown:

 

  • I used a Local Switch for this particular example. This is because the switch will not be used anywhere else in the game. It is only unique to this Scene.
  • The Label ItemGet shows the outcome if you decide to check out the treasure chest. You can see this is where the condition rests and this is where we turn on the switch if the player has not obtained the hookshot yet.

  • The Label Ignored is if the player decided to say No to checking out the chest.
