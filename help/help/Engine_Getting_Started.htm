Getting Started: Basic Engine

 

This topic is about Visual Novel Maker's Basic-Engine, not be confused with Script Documentation which is about the actual Visual Novel engine running on top of the Basic Engine.

 

The Basic Engine or just called "Engine" provides all necessary functions for basic 2D graphic, video, audio and input processing.

 

Important Notes

The implementation of the Basic Engine may be different depending on the platform or because of future updates and fixes. At time of writing, there is only one HTML5 implementation of the Basic Engine. However, there is no guarantee, maybe for future platforms we plan to support, another implementation of the Basic Engine will be implemented and used which just provides the same API.

 

That is the main reason why the Basic Engine should not be edited and it is not recommended to patch, hack or replace/override certain classes or methods of it and also not to access any non-documented or private/protected attributes/methods/classes. Otherwise update-compatibility and cross-platform support may be lost in future. But it is up to you of course.

 

Can I access DOM, jQuery, HTML5 APIs etc. in Scripts?

First of all, it is up to you. But keep in mind that accessing DOM or jQuery are things only available if your game runs in an HTML5 environment, which is maybe not the case for certain platforms. However, at time of writing, DOM/jQuery/HTML5 is available on all platforms we support and we will try to stick with HTML5.

 

Can I access OpenGL context and/or write my own Shaders?

Yes you can, but there is no guarantee that OpenGL is used for rendering on each platform. However, at time of writing, OpenGL/WebGL is used on all supported platforms so you can access OpenGL context using gs.Graphics.context and you can write own shaders using GLSL. Check out the Effects > CircularDistortionEffect example in Script Editor to learn more about how to implement your own GLSL shader effects.

 

But just keep in mind, your game will maybe not run on platforms with no OpenGL or Shader support.
