Detailed Description

This view is the core of the JSON based view/layout system.

It takes a view-descriptor, which is a JSON structure describing the view in the same way as for Custom Views in Extension Editor, and contructs a ready-to-use view from it. It is recommended to use GSOViewController instead of directly dealing with GSOView. If you work directly with GSOView, there will be no caching mechanism which means that the view will be completely rebuilt any-time you create an instance of this class which is a time-consuming process depending on the view's content.

To construct a view from an already existing Custom View, you can use the following example:

// Get a custom view by UID
var customView = [[GSDocumentManager defaultManager] documentByUid:@"<custom view uid>"];
// Create a new view, you can just pass a zero-rect.
var view = [[GSOView alloc] initWithFrame:CGRectMakeZero()];
// Set the descriptor which is just the entire items-container of the custom view document.
[view setDescriptor:customView.items];
// Create all item-view defined by the descriptor
[view createItemViews];
// Update all item views
[view update];
// Establish connections between item-view defined by delegates, bindings, etc.
[view createItemViewConnections];
// Layouts all item-views correctly
[view layoutSubviews];
// Resizes the view to fit it's contents
[view sizeToFit];
// Now the view is ready
// ......
