Detailed Description

The project manager holds and manages the current project and provides functions to open, close, create and delete projects as well as functions to get project information such as the project location on disk and more.

Keep in mind that creating, opening, closing or deleting projects are high-level functions which are non-silent and user-blocking operations. The user will be blocked by a progress-bar window until the operation has been done.

It is not recommended to create a new instance of GSProjectManager. Instead, you can access the default project manager using defaultManager (GSProjectManager) method. It is also not recommonded to hook into the VN Maker's project-management unless you exactly know what you are doing.

Project Info-Object

The project info-object is just a plain JavaScript object holding some header-information such as the project name, gameTitle, template and filePath. You can access that object using GSProjectManager::currentProject method. The recent project list in Home View consists of project info-objects which are stored in the <install-dir>/projects/projects.json file. If you pick a recent project, that info-object is passed to openProject: (GSProjectManager) to open the project.

Project Document

The project document is just a regular GSDocument and behaves like any other document. It stores general information about the project such as project name, game title and version number. You can access the project document easily using GSProjectManager::projectDocument function or alternativly:

// Get project document using document manager.
var projectDocument = [[GSDocumentManager defaultManager] documentByType:@"project"];
// Print project name
console.log(projectDocument.items.name):
