Detailed Description

An abstract base item-view class, also known as "UI-Control", for all item views which visually present the data of an object's item/field.

If you want to create a custom item-view/UI-Control you have to derive from this class. You have to register your new item-view/UI-Control using the GSOItemViewFactory before it can be used.

An item-view can be just a simple text but also a very complex graphical view/editor. Read the "Custom Controls" section of the Help File for more informations and examples. You can find that section under "Script & Plugin Reference > Extension Documentation > Getting Started > Examples". There is also a fully documented slider-control example in Extension Editor under "Scripts > Custom Controls > EXSliderItemView".

- (void) attributeChange: (id)  anAttribute
implementation

Fires an attribute-change and transfers the specified source-attribute to all bound target item-views. For Example:

[self attributeChange: { name: "myValue", value: 100 }]

That will transfer the value 100 for all target item-views bound for "myValue" as source-field.

Parameters
anAttributeThe attribute to fire: { name, value }
- (void) attributeDidChange: (id)  aSender
attribute: (id)  anAttribute
binding: (id)  aBinding 
implementation

Needs to be override in derived classes. It is important call the base-implementation of it. This method is called whenever a source-attribute of an item-view this item-view is bound to has changed. You can implement the transfer to the target-field of the binding here. An example could like this:

  • (void)attributeDidChange:(id)aSender attribute:(id)anAttribute binding:(id)aBinding { if(aBinding.target == "myValue") { _myValue = anAttribute.value; } }

Parameters
aSenderThe source item-view.
anAttributeThe attribute which has been changed: { name, value }.
aBindingThe binding-descriptor.
