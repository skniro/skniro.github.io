Detailed Description

The game manager lets you start a test-play of your game and also package/export your game.

Call the defaultManager method to get the current instance of GSGameManager.

- (CPArray) collectDocuments
implementation

Collects all UIDs of all documents which are necessary for an exported game to run properly. Documents such as Custom View, Scene Commands, etc. are not collected since those documents are not necessary for the game to run and only required by the VN Maker editor.

You can overwrite this method to control what kind of documents you want to export. But don't forget to add "SUMMARIES" as UID too if not already present.

Returns
An array of UIDs.

Creates the default test-play build. It makes everything ready for a test-play on Windows, Linux or Mac. It is automatically called by one of the startGame methods if necessary. You should not call this method directly.

- (id) createDefaultParameters
implementation

Creates a parameters object with default values which is ready to pass to one of the startGame method.

The parameters object is accessible in-game via $PARAMS object. It is recommended to call this method to get the default parameters and then add your own if you need.

Returns
The parameters object filled with default values.

Creates a game package at the specified output location. The build-configuration is taken from the packageInfo object. Read more about this in the "Build & Run" section of the Help file.

You can overwrite this method to take full control over the game-export

.

Parameters
outputPathThe location where the raw game package should be created.
parametersThe parameters-object accessible via global $PARAMS variable in-game. If you are not sure what parameters are necessary, use createPackageParameters to create a default parameters object for your package.
idsThe UIDs of the documents you want to export. See collectDocuments method for more information.
packageInfoThe package-info structure for create game package dialog-box.
aDidEndBlockA block called if the game package as been create

- (void) createResourceIndex
implementation

Creates/Refreshes the resources-index (data/RESOURCES.json) for test-play. That automatically happens if you start the game via one of the startGame methods. But if you add or change resources programatically, you have to call that method to make sure that Live Preview will be up to date. Otherwise the new/changed resources are not accessible for Live Preview until a regular play-test has been done.

- (void) createScripts: (id)  parameters
implementation

Creates/Refreshes the scripts in development-folder of the project to make latest script-changes available for test-play. That automatically happens if you start the game via one of the startGame methods. But if you add or change scripts programatically, you have to call that method to make sure that Live Preview will be up to date. Otherwise the new/changed scripts are not used by Live Preview until a regular test-play has been done.
