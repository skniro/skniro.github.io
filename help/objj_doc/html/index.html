Main Page

VN Maker API documentation describes all exposed Objective-J classes and functions for VN Maker Editor-Plugin development. Since VN Maker uses the Cappuccino Framework and the Objective-J programming language, it is recommended to check out the Cappuccino Website and Cappuccino API Documentation as well.

Developing plugins using Objective-J and the VN Maker API is only necessary for more complex plugins such as custom UI controls in most cases. For many other plugins, it is enough to work with JSON based views and JavaScript only which is much more easy. Check out the "Script & Plugin Reference > Extension Documentation" in VN Maker's Help File for more information about how to make Plugins for VN Maker in general.

How to start?

First of all, it is recommended to have at least a little bit knowledge about Desktop Application Development and/or traditional GUI development to understand things better.

You should first learn about the Document Management System (DMS) to get the knowledge how to store and access data. Check out GSDocument, GSDocumentManager and GSDatabase classes first to get an idea of the DMS. After that, you should take a look at the Resource Management System (RMS) to learn how to load and access resources such as images, audio, etc. Check out GSResource, GSResourceManager to get an idea of it.

Next step should be (custom) item-views, also called "Custom Controls", which are used to built views and windows in Extension Editor using JSON definitions. So yes, you can make your own custom control types and use them in JSON the same way as the internal ones. Checkout GSOItemView, GSODialog and all the osther GSO classes to learn more about it and also check out the "Custom Controls" section in "Script & Plugin Reference > Extension Document" of the Help File.

Hidden / Undocumented Classes & Functions

You will maybe figure out that there are a lot of hidden/undocumented classes and functions in VN Maker since the VN Maker API documentation only covers a very small amount of the entire system. Please refrain from accessing and/or reverse-engineering those classes and functions since they are not made for external access and can bring VN Maker in an unstable state if used incorrectly.

If you have the feeling that certain hidden/undocumented classes or functions should be exposed and added to the VN Maker API, don't hesitate to visit our official Forums, we are open to discuss such cases and maybe extend the VN Maker API if it makes sense. Sometimes, we may be just missed to add and describe certain classes or functions.

What if I need help? Any more examples or tutorials?

If you need help or if the documentation doesn't contain the information you want to know, don't hesitate to visit our official Forums anytime and create a topic in the appropriate category. Use the search function to see if your question maybe already has been asked by someone else.

It also highly recommended to read the "Script & Plugin Reference" in the Help File, where you also can find a lot of examples. If you are more interesting in how to deal with the Basic Engine, especially since that engine can be used in the Editor as well, you should check out "Scripter's Guide" in Help File too.

Please have fun developing plugins for VN Maker!
