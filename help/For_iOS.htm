For iOS

 

Testing & Deployment for iOS requires a few preparation steps to work properly due to the much higher security standards of the iOS platform. This topic will guide you through the necessary steps and tries to make everything as easy as possible for you. If you get stuck or have any question, please take a look at our support forums.

 

Requirements

 

To test and/or deploy your game on iOS, you have to meet the following requirements:

 

  • An Apple Developer Account + iOS Developer Membership.
  • A computer with macOS.
  • An iOS Device such as an iPhone or iPad
  • Xcode Development Tools
  • Node.js
  • Cordova

 

Let's go through them step-by-step.

 

Become an Apple Developer for iOS

 

Becoming an Apple Developer for iOS is very easy. All what you have todo is to visit the Apple Developer website, which is http://developer.apple.com and register an account or just use your already existing Apple ID.

 

After you logged in, follow the website and join the iOS Developer Program which will cost you 99 USD per year. Within the next step, you have to create a provisioning profile, register your device as a developer device and download & install the provisioning profile on your computer. Just follow the guidelines on the developer website to set everything up.

 

Install Xcode Development Tools

 

You need the Xcode Development Tools to build and deploy your game for iOS. Just login into the Apple Developer Portal and check out the downloads section which usually is: https://developer.apple.com/download/more/

 

Install Node.js

 

You need Node.js to run Cordova which is needed to package your game correctly for iOS deployment. You can download Node.js from http://nodejs.org

 

Install Cordova

 

You need Cordova to package your game correctly for iOS deployment. To install Cordova, just open a Terminal. If you never worked with a Terminal on macOS before, don't worry, you can easily find it if you just type "Terminal" in spot-light search or launch-pad.

 

Once the terminal has been opened, you will see a black window with a blinking cursor where you can type in some text (commands). Just type the following command:

 

npm install -g cordova

 

and press the return-key. After that, a lot of text will be displayed on terminal window. Just wait until everything is done. Then type:

 

cordova --version

 

and press return-key. If you see the cordova version number, the installation was successful. If you get an error, something went wrong. In that case, take a look at the text/message on the terminal window and take a look in our forums for help.

 

Prepare your VN Maker Project for iOS Deployment

 

Since everything has been setup now, you can start to build your game for iOS! Make sure your project has a game title in Database > System.

 

Audio & Video Assets

 

At the time of writing, iOS only supports the .m4a/.mp4/.mp3/.aac/.wav audio formats and .mp4 video formats. If you have used another format such as .ogg or .webm, make sure that you convert all your audio and video files into the necessary format. VN Maker allows you to import the same file multiple times with a different format and then automatically chooses the right format at runtime which is useful for HTML5 Webbrowser Games. However, if you plan to ship your game offline over the App Store, you should delete any .ogg or .webm files to reduce file size.

 

For video files, due to technical restrictions the video-playback is only supported for iOS 10+.

 

Project Preferences

 

The last step  you need to do is to open the Project Preferences and navigate to the iOS Deployment tab.

(The displayed provisioning profile and development team are not real and random just to give you an idea what kind of data you have to input here)

 

To get the provisioning profile, the easiest way is to open your downloaded provisioning profile file (.mobileprovision) with a text editor such as TextEdit and then look for UUID and a long hex-number such as displayed on the screenshot aboive.

 

You can find the Development Team Identifier in the provisioning profile file as well. Just look for com.apple.developer.team-identifier. Also, make sure to enter a unique Game ID on General tab.

 

Once you have entered all the necessary information, click the Close button and save your project.

 

 

Testing on iOS

 

If you want to do a quick play-test on iOS, just connect your device to your computer and make sure it stays unlocked/active.

 

Next choose Build For > iOS (Cordova) from The Menubar to build your game for iOS. The build-process will take a few minutes. After the build-process is done, open the Debug Console via Tools > Debug Console and check for any errors. Look for status message such as "**ARCHIVE_SUCCEEDED**" or "**EXPORT_SUCCEEDED**" to make sure the build-process went through without any errors.

 

You can find the .ipa file inside the build-folder of your project:

 

<project-folder>/build/<project-name>/<project-name>/platforms/ios/build/device/<project-name>.ipa

 

if you want to install it manually.

 

Now you can play your game on your connected device by choosing Play On > iOS (Cordova) from The Menubar. You game-build will be transfered and installed to your connected device and then start automatically. Check Debug Console for any errors.

 

Deploy for iOS

 

You can deploy your game for iOS using the Create Game Package wizard. Just select "iOS (Cordova)" from the list of platforms. Open the Debug Console to check for errors. The package creation may take a while. You can find the .ipa file at:

 

<output-location>/<project-name>/<project-name>/platforms/ios/build/device/<project-name>.ipa.

 

For more information about how to submit your app to the App Store, just follow the guides on the Apple Developer Website.

 

Troubleshooting

 

If you went through all the above steps without any errors then congratulations! But that is not the regular case, especially not if your doing this the first time. So don't worry if you get stuck or get any error messages since those are related to wrong configuration in most cases and can be solved easily.

 

The first thing you should try to do is to figure out the error message by checking the error logs. Sometimes the error messages are understandable even my non-programmers. Common errors are:

 

  • Your device is not registered for the provisioning profile
  • Your device is not connected or on lock-screen
  • The provisioning profile and the team identifier doesn't match
  • The provisioning profile and/or the team identifier are wrong
  • The provisioning profile has been expired
  • The code sign identity is wrong
  • Cordova has not been installed
  • Xcode Command Line Tools are not installed
  • Audio and/or Video files have the wrong format

 

If you get stuck with any error and if you cannot fix them yourself, just take a look at our support forums.
