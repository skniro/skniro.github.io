Web用

 

Webプラットフォームの場合、テストとデプロイは非常に簡単です。必要な手順について説明します。

 

Webプラットフォームでのテスト

 

簡単なテストビルドを行うには、メニューバーから「ゲーム → 指定してビルド → Web Browser」を選択します。その後、プロジェクトフォルダを選択するか、メニューバーから「ゲーム → ゲームフォルダを開く」を選択します。フォルダにはいくつかのファイルとindex.htmlファイルが1つ入っています。このファイルを Firefoxで開くとゲームをプレイできます。

 

他にも、実際の環境に近い状態でテストを行うために、ローカルのWebサーバーを設定したり、ビルドフォルダをWebサーバーにアップロードしたりするなどの方法もあります。

 

オーディオと動画のアセット

 

現時点で、Webブラウザの半数はオーディオ形式として.m4a／.mp4／.mp3／.aac／.wav、動画形式として.mp4のみに対応となっています。残りの半数はオーディオ形式として.ogg／.wav、動画形式として.webmのみ対応です。主要なブラウザをすべてサポートしたい場合、すべてのオーディオファイルと動画ファイルを、2つ以上の形式でゲームに含める必要があります。シンプルな効果音用の.wavファイルは唯一の例外となります。

 

ラノゲツクールMVでは、同じファイルを異なる形式で複数回インポートし、ランタイム時に適切な形式を選択できます。

 

注：一部のモバイルWebブラウザでは、セキュリティ上の制限により、動画再生が正しく機能しない場合があります。

 

Web用開始ロゴ

特にモバイルの一部のプラットフォームでは、モバイル版のWebブラウザでゲームをロードすると、Web用開始画面が表示されます。ゲームを実際に開始するには、プレイヤーがこの画面をタップする必要があります。この画面は、モバイルブラウザのセキュリティ要件が厳しいことが理由で設定されています。

 

 

「リソースマネージャー」を開き、「Graphics/Pictures」フォルダにある「web-start-screen」画像ファイルを差し替えることで、Web用開始ロゴを変更することができます。

 

Webプラットフォームでのデプロイ

 

ゲームをデプロイする場合、メニューバーから「ゲーム　→　指定してビルド」を選択する方法か「ゲームパッケージを作成」ウィザードを使用してすぐに提供可能なWebパッケージを作成する方法を選択できます。後は、Web用パッケージをWebサーバーにアップロードするだけで、全世界のユーザーに公開できるようになりました！

 

ゲームのパッケージを詳細に設定する方法については、「ビルドと起動」のページを参照してください。
