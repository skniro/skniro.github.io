iOS用

 

iOSプラットフォームには非常に高いセキュリティ基準が設けられているため、iOSでのテストとデプロイを正しく機能させるにはいくつかの準備手順が必要です。必要な手順について説明します。疑問や不明な点がある場合は、当社のサポートフォーラムを参照してください。

 

要件

 

iOSでゲームをテスト／デプロイするには次の要件を満たす必要があります。

 

  • Apple Developer AccountとiOS Developer Membership
  • macOS搭載のコンピュータ
  • iOS端末（iPhone、iPadなど）
  • Xcode開発ツール
  • Node.js
  • Cordova

 

各要件の説明は次の通りです。

 

Apple Developer for iOSになる

 

Apple Developer for iOSになるのは非常に簡単です。 まずはApple Developerのウェブサイト（http://developer.apple.com）にアクセスしてアカウントを登録するか、既存のApple IDを使用します。

 

ログイン後、ウェブサイトの指示に従ってiOS Developer Programに参加します。次に、プロビジョニングプロフィールを作成し、端末を開発者端末として登録し、コンピューターにプロビジョニングプロフィールをダウンロードしてインストールします。開発者ウェブサイトのガイドラインに従い、すべての設定手順を完了してください。

 

Xcode開発ツールをインストールする

 

ゲームをビルドしてiOSにデプロイするにはXcode開発ツールが必要です。Apple Developer Portalにログインし、ダウンロードセクション を確認してください。

 

Node.jsをインストールする

 

iOSへのデプロイ用にゲームをパッケージングするのに必要なCordovaを起動するにはNode.jsが必要です。Node.jsは対象サイトからダウンロードできます。

 

Cordovaをインストールする

 

iOSへのデプロイ用にゲームをパッケージングするにはCordovaが必要です。Cordovaをインストールするにはターミナルを開きます。macOSで、スポットライト検索やLaunchpadに「Terminal」と入力してください。

 

ターミナルが開くと黒いウィンドウが表示されます。カーソルが点滅している箇所にはテキスト（コマンド）を入力できます。次のコマンドを入力してください。

 

npm install -g cordova

 

入力後にEnterキーを押すと、ターミナルウィンドウにメッセージが表示されます。処理が完了するのを待ってから次のコマンドを入力してください。

 

cordova --version

 

入力後、Enterキーを押します。Cordovaのバージョン番号が表示された場合、インストールは成功です。エラーが表示された場合は問題が生じています。その場合は、ターミナルウィンドウのテキスト／メッセージを確認し、当社フォーラムを参照してください。

 

ラノゲツクールMVプロジェクトをiOSへのデプロイ用に準備する

 

すべての設定が完了し、iOS用にゲームをビルドできるようになりました。「データベース → システム」でプロジェクトにゲームタイトルが設定されていることを確認してください。

 

オーディオと動画のアセット

 

現時点で、iOSはオーディオ形式として.m4a／.mp4／.mp3／.aac／.wav、動画形式として.mp4のみ対応しています。.oggや.webmvideoなどの別の形式を使用している場合は、オーディオや動画ファイルを適切な形式に変換してください。VN Makerでは、同じファイルを異なる形式で複数回インポートし、ランタイム時に適切な形式を選択できます。この方法はHTML5ウェブブラウザゲームで便利です。ただし、App Storeでゲームをオフライン提供する場合は、.oggファイルや.webmファイルをすべて削除してファイルサイズを抑える必要があります。

動画ファイルについては、技術的制限により、iOS 10+以外の動画再生はサポートされていません。

 

プロジェクト環境設定

 

最後の手順では、「プロジェクト環境設定」を開き、「iOS Deployment」タブを選択します。

（上のスクリーンショットのプロビジョニングプロフィールと開発チームは一例で、実際には作成者が入力したデータが表示されます）

 

"プロビジョニングプロフィールを取得する最も簡単な方法は、ダウンロードしたプロビジョニングプロフィールファイル（.mobileprovision）をテキストエディター（TextEditなど）で開き、 その中に記載されているUUIDと16進数を確認することでしょう。"

 

プロビジョニングプロフィールファイルには開発チームIDも表示されます。com.apple.developer.team-identifierを確認してください。全般タブで、ゲームのユニークなゲームIDを設定することを忘れないでください。必要な情報をすべて入力したら、「閉じる」ボタンをクリックしてプロジェクトを保存します。

 

 

iOSでのテスト

 

iOSで簡単なプレイテストを行うには、端末をコンピュータに接続し、端末がロック解除されていること／アクティブであることを確認します。

 

次はメニューバーから「指定してビルド → iOS（Cordova）」を選択し、iOS用にゲームをビルドします。ビルドプロセスは数分かかります。ビルドプロセスがの完了後、「ツール → デバッグコンソール 」からデバッグコンソールを開き、エラーがないかどうか点検します。「**ARCHIVE_SUCCEEDED**」（アーカイブ完了）や「**EXPORT_SUCCEEDED**」（エクスポート完了）などのメッセージを探し、ビルドプロセスが正常に完了していることを確認してください。

 

プロジェクトのビルドフォルダには、.ipa ファイルがあります。

 

<プロジェクトフォルダ>/build/<プロジェクト名>/<プロジェクト名>/platforms/ios/build/device/<プロジェクト名>.ipa

 

上記の場所から必要に応じて.ipa ファイルを手動でインストールできます。

 

これでメニューバーから「指定してプレイ → iOS（Cordova）」を選択すると、接続した端末でゲームをプレイできます。ゲームのビルドは接続された端末に転送およびインストールされ、自動的に起動します。デバッグコンソールでエラーがないかどうか点検してください。

 

iOSへのデプロイ

 

「ゲームパッケージを作成」ウィザードを使用すると、iOSにゲームをデプロイできます。プラットフォームのリストから「iOS (Cordova)」を選択し、デバッグコンソールを開いてエラーがないかどうか点検してください。パッケージの作成には時間がかかる場合があります。.ipaファイルは以下の場所にあります。

 

<出力場所>/<プロジェクト名>/<プロジェクト名>/platforms/ios/build/device/<プロジェクト名>.ipa.

 

アプリをApp Storeで公開する方法の詳細については、Apple Developer ウェブサイトのガイドに従ってください。

 

トラブルシューティング

 

上記手順がすべてエラーなしに終わればテストとデプロイは完了ですが、特にゲームのテストとデプロイを初めて行う場合、エラーなしに終わることはまれです。エラーメッセージが表示されても心配はありません。設定の誤りに関連することがほとんどで、簡単に解決できます。

 

トラブルシューティングの最初の手順はエラーログを点検してエラーメッセージを把握することです。一部のエラーメッセージはプログラマーでなくても理解できるものになっています。以下に一般的なエラーを示します。

 

  • 端末がプロビジョニングプロフィールに登録されていません
  • 端末が接続されていないかロック画面になっています
  • プロビジョニングプロフィールとチームIDが一致しません
  • プロビジョニングプロフィールかチームIDまたはその両方が間違っています
  • プロビジョニングプロフィールの期限が切れています
  • コード署名IDが間違っています
  • Cordovaがインストールされていません
  • Xcodeコマンドラインツールがインストールされていません
  • オーディオファイルか動画ファイルの形式が間違っています

 

エラーを解決できない場合は、当社のサポートフォーラムを参照してください。
