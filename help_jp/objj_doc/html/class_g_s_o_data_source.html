Detailed Description

This class handles data-sources defined by data-source descriptors, usually defined in JSON based views/commands, and easily allows to fetch the data from it.

The term "item" is used to describe a single element fetched from the backend-source like a single document. The term "item-definition" is used to describe an item-definition in a database-descriptor like the following:

...
dataSource: [{ name: "Item 1", alias: "item1" }, {name: "Item 2", alias: "item1"} ]
...

If an item is requested from a data-source which uses item-definitions, the item represents the item-definition object itself. If no alias-value is present, the index in the array of item-definitions is used instead.
