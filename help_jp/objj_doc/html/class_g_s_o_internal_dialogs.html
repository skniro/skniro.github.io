GSOInternalDialogs Class Reference

Detailed Description

An enum of all internal dialog which can be easily triggered using GSOWindowController or GSODialog class to show up commonly used dialog-boxes such as graphic, audio, video or variable selection dialog-boxes.

dialog.selectGraphicResource
Shows graphic resource selection dialog-box. The following parameters are accepted:

  • folder - The resource folder to display, for example: Graphics/Pictures.
  • emptySelection - Indicates if "(None)" selection is allowed.
  • multiSelection - Indicates if multiple resources can be selected.

dialog.selectVideoResource
Shows video resource selection dialog-box. The following parameters are accepted:

  • folder - The resource folder to display, for example: Graphics/Pictures.
  • emptySelection - Indicates if "(None)" selection is allowed.
  • multiSelection - Indicates if multiple resources can be selected.

dialog.selectAudioResource
Shows audio resource selection dialog-box. The following parameters are accepted:

  • folder - The resource folder to display, for example: Graphics/Pictures.
  • emptySelection - Indicates if "(None)" selection is allowed.
  • multiSelection - Indicates if multiple resources can be selected.

dialog.selectVariable
Shows variable selection dialog-box. The following parameters are accepted:

  • dataSource - Kind of variables to select. Can be "numbers", "booleans", "strings" or "lists".

dialog.selectParameters
Shows parameter selection dialog-box which is used for Common Events for example. You have to specify the list parameters, in the format as defined in Common Events - Parameters Section, via GSOItemView::globalData by adding a "parameters" property to it. For Example:

[GSOItemView globalData].parameters = [
{
"label": "My Value:",
"type": 1,
"numberVariable": {
"index": 0,
"scope": 0
},
"numberValueType": "number"
},
{
"label": "My Image:",
"type": 3,
"stringVariable": {
"index": 0,
"scope": 0
},
"stringValueType": "Graphics/Pictures"
}
];
var dialog = [GSODialog createDialog:{ uid: "dialog.selectParameters" }];
[dialog showModalWindowWithTarget:self didEndSelector:@selector(dialogDidEnd:code:)];

Check out Common Events Data-Record View in Extension Editor to get more information about the data-format for parameters.
