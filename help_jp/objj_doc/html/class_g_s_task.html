Detailed Description

A task handles a single piece of blocking-work which needs to be done such as: Compile scripts, save project, create game package, etc.

In regular, the user cannot continue working with VN Maker until the task has been finished. Check out the GSTaskManager for more information.

This class is the base-class of all tasks in VN Maker. So to implement your own task, you have to derive a new class from this one and implement the necessary methods. In regular, you only need to implement the "run" method.

@implementation CustomTask : GSTask
{
}
- (id)init
{
self = [super init];
[self setDescription:@"My Custom Task"];
return self;
}
- (void)run
{
// Task Logic Here.
}
@end

- (void) setDelegate: (CPObject) 
implementation

Sets the delegate.

The following methods are called on the delegate if implemented:

  • task:(GSTask)aTask statusDidChange:(CPInteger)aStatus - Called if task status did change.
  • task:(GSTask)aTask detailsDidChange:(CPInteger)details - Called if task details-message did change.
  • task:(GSTask)aTask descriptionDidChange:(CPInteger)aDescription - Called if task description message did change.
  • task:(GSTask)aTask errorDidOccur:(id)errorData - Called if an error did occur during task execution.
  • taskDidFinish:(GSTask)aTask withResult:(id)resultData - Called if the task has been finished.
