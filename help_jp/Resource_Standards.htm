リソースの基準

 

ラノゲツクールMVでは、ユーザー固有のリソースをインポートすることができます。指定しない限り、アセットにかかわらず、画像サイズの基準はありません。モバイルゲームの場合は、1024px ~ 2048pxの範囲内に収めることをお勧めします。

 

「キャラクター」

キャラクターの表情に使用される画像です。

  • データ形式 - PNG

  • アニメーション - フレームごとの画像が必要です。

 

「背景」

背景に使用される画像です。

  • データ形式 - PNG、JPG

 

「ピクチャー」

CG、アニメーション、システム、画像、UIコマンドで使用される画像です。

  • データ形式 - PNG、JPG

  • 画像サイズ：

  • CGギャラリーのサムネイル：ギャラリーサムネイルのサイズは画面解像度の1／8です。たとえば、画面解像度が1280x720の場合は160x90になります。

  • アニメーション：スプライトシートとしてインポートされている必要があります。サイズは画像のグリッドサイズによって異なります。以下に例を示します。

    キャラクターのグリッドが幅（X）24  x 高さ（Y）32の場合、次のような計算になります。

  • （幅 x フレーム）px x （高さ x フレーム）px
    幅 24 x 3フレーム = 72
    高さ 32 x 4フレーム = 128
    最終的な画像サイズ 72px x 128px

 

「フレーム」

フレームは画面上にグラフィックフレームを表示し、ゲーム内のウィンドウやカーソルなどゲーム内UIの装飾に使用します。フレームは、「 ピクチャーを表示」コマンドの「ビジュアルタイプ」が「フレーム」に設定されている場合に使用できます。

 

実際の見た目は画像（スキンとも呼ばれます）に左右されます。当然ながら、フレームだけでなくウィンドウ背景も適切なものを選ぶことでより見栄えよく仕上げることができます。上の例でスキン画像をインポートすると次のようになります。

黄色でマークされた領域はコーナー、赤でマークされた領域はサイドと呼ばれ、フレームのサイズに適合するように延長されます。コーナーとサイドのサイズは、「ピクチャーを表示」コマンドのフィールドから設定できます。

 

「3-パーツイメージ」

3-パーツイメージは、フレームよりもシンプルなフォームで、画像を3つの断片に分けて個別に表示します。「ピクチャーを表示」コマンドの「ビジュアルタイプ」が「3-パーツイメージ」に設定されている場合に使用できます。

「アイコン」

「言語選択」ウィンドウで使用される画像です。

  • データ形式 - PNG、JPG

 

 

「マスク」

マスクで使用される画像です。

  • データ形式 - PNG、JPG

 

「フォント」

ゲームに使用されるフォントです。

  • データ形式 - WOFF、FNT

  • WOFFフォント：WOFFerなどのツールを使用すると、TTF ファイルをWOFFに簡単に変換できます。

  • bitmapフォント：bmfontフォント、bmGlyphフォントの他、フォントジェネレーターがエクスポートするfnt/bmfont形式がサポートされます。ファイル形式はfnt-textである必要があります。XML/Binaryはサポートされていません。bmGlyphを使用する場合は、出力形式に「bmfont」を選択してください。フォーマットは、テキストの色の変更以外、bitmapフォントには影響しません。

  • 重要事項：bitmapフォントは、文字セット、スタイル、サイズ、使用されるbitmapフォントの数によって大量のメモリを消費する場合があります。特にスマートフォンなどの携帯端末でbitmapフォントを使用する場合は注意が必要です。

 

「音楽と効果音」

ゲームで使用されるオーディオトラックです。

  • データ形式 - OGG、M4A

 

「Live2D」

Live2D用のファイルです。

  • データ形式 - Live2Dモデル、モーション、顔の表情は、インポートプロセス中に自動的に.live2dアーカイブ（.zip）にパッケージングされます。Live2Dモデルをインポートするには、次のスクリーンショットのように.model.jsonを選択する必要があります。

 

 

Live2Dモデルは、ほとんどがJSON形式です。もしJSONファイルではなく、生のモデルファイルしか持っていない場合は、Live2D Viewerと呼ばれるソフトを使い、自身のLive2DモデルをJSON形式でエクスポートしてください。JSONファイルとしてエクスポートする前に、表情やモーション、物理、ポーズといったファイルがすべて正しく追加されていることを確認しておきましょう。

JSON file.

 

 

メニューバーから「 プロジェクト → 書き出し → モデル設定ファイル 」と選択し、JSON形式でエクスポートしましょう。
