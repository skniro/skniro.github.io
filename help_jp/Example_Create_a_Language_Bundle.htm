例：言語バンドルの作成

 

この例では、エクステンションを他の言語に翻訳する言語バンドルを作成します。そのためには、「Language Bundles」を右クリックして、「新規」 → 「言語バンドル」の順に選択します。

これを「CardExtension」と名付けます。

 

{ "phrases": { "D63591732C7F724D8D98B6E5B34B914FFFD6": "Sample Text" } }

 

これにはフレーズ例が含まれているので、削除します。次に、別の言語バンドルを作成して「CardExtension.de」と名付けます。

「CardExtension」バンドルには、既定の言語でフレーズが含まれています。現在の言語での翻訳がない場合、既定の言語が使用されます。

2つ目のバンドル「CardExtension.de」は、ドイツ語を表す「.de」が接尾辞として付けられています。つまり現在の言語がドイツ語の場合、このバンドルが使用されます。

 

ではプロパティを見てみましょう。

 

  • phrases

バンドルの異なるフレーズを定義する、キーフレーズのペアを含みます。つまり、各フレーズには固有のキーが必要になります。キーは、固有でさえあればどんなテキストでも構いません。

既定では、各フレーズにはUIDが使われています。UIビューでフレーズを使用するには、「$$CardExtension.D63591732C7F724D8D98B6E5B34B914FFFD6」のような、

$$<bundle>.<phrase_uid>構文を使用することができます。このような構文は、表示可能なテキストを指定できるほとんどの場所で使うことができます。以下に例を示します。

 

{

    "type": "GSLabel",

    "text": "$$CardExtension.D63591732C7F724D8D98B6E5B34B914FFFD6",

    "frame": [0, 0]  

}

 

このラベルは、存在する場合、「D63591732C7F724D8D98B6E5B34B914FFFD6」キーのフレーズを表示します。

はじめこのフレーズは現在の言語用に存在しているように見えますが、そうでない場合、既定の言語によるフレーズを使用します。

 

新しいフレーズを作成するには、手動でバンドルに加えればいいのですが、作業量が増えると少々厄介に感じられるので、自動でこれを行ってくれる2つの便利な関数があります。

その2つとは、「Insert New String」と「Insert New String (LCS)」です。これらの関数は「ツール」 → 「デベロッパーツール」のメニューバーまたは

エクステンションエディタのツールバーにあります。

 

どちらの関数も実際には同じことをしますが、出力時のフォーマットが異なります。これらの関数がどのように便利なのか、確認してみましょう。

 

まず、ビュー、シーンコマンド、スクリプトのいずれかで、翻訳するテキストを選択する必要があります。次に、「Insert New String」をクリックします。

するとバンドル名を入力するようメッセージが表示されます。バンドル「CardExtension」の名前を入力したら「OK」をクリックします。

すると、次の2つのことが起こります。

 

1．選択したテキストが自動的に「$$CardExtension.<uid>」に置き換わります。

2．選択したテキストがデフォルトの言語の「CardExtension」バンドルに追加され、UIDが自動的に生成されます。

 

これで今後は何かを翻訳する際に毎回自分の手でUIDを作って手動でフレーズを追加しなくてもよくなるので、時間を大幅に節約できます。

この2つの便利な関数は1つだけ使いましょう。もう1つの関数「Insert New String (LCS)」の動作は同じですが、出力は次のとおりになります。

 

lcs('<bundle>.<uid>')

 

これは必要に応じ、スクリプトや計算式で使用することができます。
