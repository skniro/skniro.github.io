はじめに：エクステンション


ラノゲツクールMVには、エディターやゲームエンジン向けに新たな機能を開発して他の人々と共有できる強力なエクステンションシステムがあります。

エクステンションを開発するには、「表示」 → 「エクステンション」を開いてください。

 

 

エクステンションドキュメント

左側にプロジェクトのすべての「エクステンションドキュメント」があります。エクステンションドキュメントはフォルダーを使って自由に整理することができます。

プロジェクトを保存するたび、すべての編集済みのエクステンションドキュメントが内部JavaScriptオブジェクトにコンパイルされ、構文エラーに関するメッセージが表示されます。

構文エラーが発生したら、該当するエラーを修正し、もう一度プロジェクトを保存してください。

 

スクリプト/コードエディタ

右側には現在選択されているエクステンションドキュメントの内容を表示する「コードエディタ」があります。ラノゲツクールMVはACEコードエディタを使用しています。

ACEそのものについて詳しくは、www.ace.c9.io を参照してください。フォントサイズなどを変更できるACE設定を開くには、Ctrl + ,（コンマ）またはCmd + ,（コンマ）を使用します。

 

エクステンションの作成

ラノゲツクールMVに新しい機能を追加したい場合は、まずはこのガイドを利用するといいでしょう。

さらなるサポートが必要な場合は、プラグインAPIリファレンスや当社のフォーラムをチェックしてみてください。

 

エクステンションを作成するには、「ツール」 → 「デベロッパーツール」 → 「エクステンションを作成」でエクステンションコンポーザーを開きます。

 

ただし、エクステンションを作成する前に、エディターまたはゲームエンジンに変更を加えてエクステンションを開発する必要があります。

 

ゲームエンジンの修正

ゲームエンジンに変更を加えるには、「表示」 → 「スクリプト」のメニューからスクリプトエディタを開きます。ここでは変更や削除が可能なすべてのゲームスクリプトが確認できます。

スクリプトやクラスを拡張する場合、特定のスクリプトに直接変更を加えるのではなく、新規のスクリプトを作成することをおすすめします。

そうしなければ、エクステンション間でコリジョンが起こることがあります。

 

ゲームエディタの修正

ゲームエディタに変更を加えるには、「表示」 → 「エクステンション」を開きます。

 

エクステンションエディタでは、次のエディター要素を作成または変更することができます。

 

  • データレコードビュー

データベースのカテゴリーを説明します。1つのカテゴリーデータレコードについて、名前、バックエンドフィールド、UI全体が含まれています。

「キャラクター」など、すべての既存のカテゴリーを確認、変更、または削除することができます。
ただし、「"System " DataRecordView」は技術的な理由により、必ずこのビュー内に存在する必要があるため、削除することはおすすめしません。

 

  • シーンコマンド

ユーザーがシーンのロジックを定義するのに使用できる、シーンビューの1つのコマンドを説明します。つまり、ここで新しいコマンドを追加したり、既存のコマンドを変更、削除することもできます。

 

  • カスタムビュー

他のビューやシーンコマンドによって、さまざまな方法で使用することのできる独立したビューです。例えば、ポップアップコントロールのポップアップダイアログとして使用したり、

パーシャルビューとして別のビューに埋め込むのに使用したりすることが可能です。

 

  • JavaScriptファイル

定数を定義するためのビューまたはシーンコマンドで使用されます。ランタイムでテキストやその他の種類のロジックを算出したり、デザイン時間中にJSONを作成したりします。

 

  • Objective-Jファイル

カスタムUIのコントロール、リソースパイプラインプロセッサ、より強力なスクリプトエージェント、その他さらにコントロールが必要なものなどを作成する際に使用されます。ファイル名がObjective-Jのクラス名と一致していることが重要です。

 

  • 言語バンドル

ビューのUIやシーンコマンドを異なる言語に翻訳する際に使用されます。

 

JSONを使用したビューのデザイン

ビューはJSONを使用して定義されます。ですので、テキストやその他ちょっとしたロジックをランタイムで算出する時以外、プログラミングは必要ありません。セクションとUIコントロールを定義し、それらがどのようにデータを処理すべきか、そしてどのバックエンドフィールドにそのデータが書き込まれるべきかを説明すればいいだけです。

 

ビューはセクションに分けられます。各セクションには枠とタイトル、表示アイテムのリストを設定できます。アイテムとは、ユーザーがビューとインタラクトできるようにするUIコントロールです。アイテムにはテキストフィールド、チェックボックス、ラジオボタンなど、様々なタイプがあります。詳しいリストについては、プラグインAPIリファレンスを参照してください。
