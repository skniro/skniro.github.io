基本コマンド

 

基本コマンドは、ゲームを調整する際に必ず必要となるコマンド群です。

 

「ウエイト」

シーンの処理を一時的に停止します。並列実行がオンとなっている共通イベントは対象になりません。

  • 「時間」 - 次のコマンドが実行されるまでの時間です。既定ではミリ秒（1秒 = 1000ミリ秒）で実行されます。

 

「コメント」

シーンコンテンツ内にメモを作成します。このコマンドに記入した文字は、ゲーム内で実行されることがないため、安全にコマンドとコマンドの間に挿入できます。

メモ：タブ付けが必要なコマンド（「条件」など）の下に位置していなければ、右矢印キー（）でコマンドを無効化することもできます。

 

「スイッチ」

スイッチの値（ 「有効／無効」）をコントロールする際に使用します。単体のスイッチまたは複数のスイッチを選択することができ、「ローカル」「グローバル」「永続」と3つの範囲があります。

  • 「ターゲット」 - 「スイッチ」「範囲」「参照」と3種類あります。

  • 「スイッチ」は単体のスイッチを指定します。

  • 「範囲」はスイッチインデックスXからYまでを指定します。

  • 「参照」はポインターとも呼ばれ、選択した変数に設定されているスイッチの値を、このコマンドで使用する変数のIDとして設定・取得します。

  • 「範囲」 - 変更するスイッチの範囲です。

  • 「ローカル」は現在のシーンに限定されます。

  • 「グローバル」はあらゆるシーンに使用できます。

  • 「永続」はゲームのセーブとは別にデータを保持します。「ターゲット」が「範囲」に設定されている場合は使用できません。

  • 「値」 - そのスイッチに対して行われる計算を指定する機能です。

  • 「無効」 - スイッチを無効にします。

  • 「有効」 - スイッチを有効にします。

  • 「有効／無効切り替え」 - スイッチが「有効」の場合は「無効」に、「無効」の場合は「有効」になります。

 

「数値変数」

数値変数は数値を保持するために使用されます。単体の数値変数または複数の数値変数を選択することができ、「ローカル」「グローバル」「永続」と3つの範囲があります。

  • 「ターゲット」 - 変更するターゲットです。3種類あります。

  • 「変数」は単体の変数を変更します。

  • 「範囲」は変数インデックスXからYまでを変更します。

  • 「参照」はポインターとも呼ばれ、選択した変数に設定されている数値変数を、このコマンドで使用する変数のIDとして設定・取得します。

  • 「範囲」 - 変数の場所・範囲を設定します。

  • 「ローカル」は現在のシーンに限定されます。

  • 「グローバル」はあらゆるシーンに使用できます。

  • 「永続」はゲームのセーブとは別にデータを保持します。「ターゲット」が「範囲」に設定されている場合は使用できません。

  • 「操作」 - 変数に対して行われる計算を指定する項目です。

  • 「設定」 - 選択した変数に数値を割り当てます。

  • 「加算」 - 選択した変数に数値を加算します。

  • 「減算」 - 選択した変数から数値を減算します。

  • 「乗算」 - 選択した変数に数値を乗算します。

  • 「除算」 - 選択した変数を数値で除算します。

  • 「余り」 - 選択した変数を数値で剰余算し、余りを割り当てます。

  • 「ソース」 - 実行時に使用されるソース値です。

  • 「値」 - 定数の値または変数です。

  • 「乱数」 - XからYまでのランダムな値です。

  • 「参照」 - はポインターとも呼ばれ、選択した変数に設定されている数値変数を、このコマンドで使用する変数のIDとして設定・取得します。  
    詳細については、「ヒントとコツ」を参照してください。

 

メモ：「値」に「変数」を使用するには、「...」ボタンを押します。

 

「10進型変数」

10進型変数は数値変数と同じく、数値を保持するために使用されます。単体の10進型変数と複数の10進型変数を選択することができ、「ローカル」「グローバル」「永続」と3つの範囲があります。これは、新しい種類の変数というわけではありません。「数値変数」コマンドと「10進型変数」コマンドは、どちらも同じく数値変数に対して機能します。

 

「数値変数」コマンドとの唯一の違いとして、このコマンドでは、結果の端数の処理方法を設定することが可能で、さらに値には小数点や浮動小数点が入力できます。既定では、結果の端数は処理されません。「数値変数」コマンドでは除算の結果は必ず切り捨てされ、小数点／浮動小数点を入力することができません。

 

まとめると、「10進型変数」コマンドは、小数点／浮動小数点を持つ数値を扱うためのものです。できる限り「数値変数」コマンドを使用することをお勧めします。

 

  • 「ターゲット」 - 変更するターゲットです。

  • 「変数」は単体の変数を変更します。

  • 「範囲」は変数インデックスXからYまでを変更します。

  • 「参照」はポインターとも呼ばれ、選択した変数に設定されている数値変数を、このコマンドで使用する変数のIDとして設定・取得します。  

  • 「範囲」 - 変数の場所・範囲を設定します。

  • 「ローカル」は現在のシーンに限定されます。

  • 「グローバル」はあらゆるシーンに使用できます。

  • 「永続」はゲームのセーブとは別にデータを保持します。「ターゲット」が「範囲」に設定されている場合は使用できません。

  • 「操作」 - 変数に対して行われる計算を指定するオプションです。

  • 「設定」 - 選択した変数に数値を割り当てます。

  • 「加算」 - 選択した変数に数値を加算します。

  • 「減算」 - 選択した変数から数値を減算します。

  • 「乗算」 - 選択した変数に数値を乗算します。

  • 「除算」 - 選択した変数を数値で除算します。

  • 「余り」 - 選択した変数を数値で剰余算し、余りを割り当てます。

  • 「ソース」- 実行時に使用されるソース値です。

  • 「値」 - 定数の値または変数です。

  • 「乱数」 - XからYまでのランダムな値です。

  • 「参照」 - はポインターとも呼ばれ、選択した変数に設定されている数値変数を、このコマンドで使用する変数のIDとして設定・取得します。

  • 「端数処理」 - 結果の端数処理の有無や方法を決定します。

  • 「いいえ」 - 結果は端数処理されません。

  • 「四捨五入」 - 結果は四捨五入されます。

  • 「切り上げ」 - 結果は切り上げられます。

  • 「切り捨て」 - 結果は切り捨てられます。

メモ：「値」に「変数」を使用するには、「...」ボタンを押します。

 

「テキスト変数」

テキストを保持するために使用されます。単体のテキスト変数と複数のテキスト変数を選択することができ、「ローカル」「グローバル」「永続」と3つの範囲があります。

  • 「ターゲット」 - 「テキスト」「範囲」「参照」と3種類あります。.

  • 「変数」は単体の変数を変更します。

  • 「範囲」は変数インデックスXからYまでを変更します。

  • 「参照」 - はポインターとも呼ばれ、選択した変数に設定されている数値変数を、このコマンドで使用する変数のIDとして設定・取得します。

  • 「テキスト」 - 変更する変数の場所・範囲を設定します。

  • 「ローカル」は現在のシーンに限定されます。

  • 「グローバル」はあらゆるシーンに使用できます。

  • 「永続」はゲームのセーブとは別にデータを保持します。「ターゲット」が「範囲」に設定されている場合は使用できません。

  •  「操作」 - 変数に対して行われる計算を指定するオプションです。

  • 「設定」 - 選択した変数にテキスト値を割り当てます。

  • 「追加」 -選択した変数にテキスト値を追加します。

  • 「大文字へ」 - 選択した変数のテキストをすべて大文字にします（例：VARIABLE）。

  • 「小文字へ」 - 選択した変数のテキストをすべて小文字にします（例：variable）。

 

メモ：「値」に「変数」を使用するには、「...」ボタンを押します。

 

「ラベル」

ラベルはマーカーの役割を果たすコマンドです。名前を設定することで、シーンはそのラベルの下部にある場面へとジャンプすることができます。一般的に、1つのシーンだけで様々な結末を生み出したい時に使用されます。

 

「ラベルに移動」

「ラベルに移動」は、名前が設定されたラベルすなわちマーカーにシーンを移動させます。

 

「条件」

このコマンドと使うと、コマンドを実行するために満たさなければならない条件を作成できます。次のように、条件コマンドの下位に設定してタブ付けする必要があります。

  • 「もし」 - 実行のために確認しなければならない条件の変数タイプを設定します。

  • 「数」は数値変数を確認します。

  • 「スイッチ」 はスイッチを確認します。

  • 「テキスト」はテキスト変数を確認します。

  • 「変数」 - 条件が確認する変数です。

  • 「が」 - 実行のために確認しなければならない条件を設定します。

  • 「＝」 - 「変数」と「値」が同じ内容である場合、条件コマンドを起動します。

  • 「≠」 - 「変数」と「値」が同じ内容でない限り、条件コマンドを起動します。

  • 「＞」 - 「変数」が「値」よりも大きい場合、条件コマンドを起動します。

  • 「≧」 - 「変数」が「値」よりも大きい、または等しい場合、条件コマンドを起動します。

  • 「＜」 - 「変数」が「値」よりも小さい場合、条件コマンドを起動します。

  • 「≦」 - 「変数」が「値」よりも小さい、または等しい場合、条件コマンドを起動します。

  • 「値」 - 条件を起動するトリガーです。
    メモ：「値」に「変数」を使用するには、「...」ボタンを押します。

 

コマンドを「条件」の傘下として追加したい場合は、次のような方法があります。

 

  1. 「条件」を選択している状態で、新しいコマンドをクリックすると、「条件」コマンドの下位に自動的に追加され、タブ付けされます。

  2. 新しいコマンドを「条件」の下位にドラッグすると、タブ付けされた状態で表示されます。

  3. 「条件」の下位に位置するコマンドを選択している状態で右矢印キー（）を押すと、タブ付けすることができます。

「条件」の傘下にあるコマンドを、削除せずにタブ付け解除するには、左矢印キー（）を押します。

 

メモ：「条件」の下位にあるコマンドがタブ付けされていない場合、「条件」は終了したものとみなされ、後続のタブ付けされたコマンドはすべて「コメント」として扱われます。

 

「その他」

「条件」がトリガーされなかった場合の結末を設定できます。次のシナリオのように、分岐条件を作成する場合に便利です。

 

「Else If 文」

「条件」がトリガーされなかった場合の「条件」セットを新たに設定できます。何らかの理由でいずれの条件もトリガーされた際に、古い条件がトリガーされないようにするために使用します。

 

「スイッチをチェック」

「条件」と同じく「スイッチ」の「有効／無効」を確認し、設定した条件と一致した場合、指定したラベルに移動します。

 

「数値をチェック」

「条件」と同じく「数値変数」が特定の値であるかどうかを確認し、設定した条件と一致した場合、指定したラベルに移動します。

 

「テキストをチェック」

「条件」と同じく「テキスト変数」が特定の値であるかどうかを確認し、設定した条件と一致した場合、指定したラベルに移動します。

 

「ループ」

「ループ」コマンドの傘下として設定されたコマンドは、繰り返し実行されるようになります。ループ傘下内に複数のループを設定することもできます。ループ処理は無限に続くため、ループを無効化したい場合は「ループを終了」コマンドを使用する必要があります。

 

メモ：以下に「ループ」コマンドの使用例を示します。

 

 

「ループを終了」

ループを終了します。

 

「タイマーを開始」

タイマーに設定した時間が経過した際に、共通イベントを呼び出したり、ラベルに移動したりすることができます。画面上に視覚的効果は描写されません。
タイマーはいずれも、現在のシーンでのみ適用されます。そのため、シーンを変更すると、タイマーはすべて無効になります。

  • 「数」 - タイマーのインデックス（ID）です。IDの異なるタイマーは、それぞれ独立したタイマーになります。

  • 「インターバル」 - タイマーのインターバルです。たとえば「500 ms」の場合、タイマーは500ミリ秒ごとにトリガーされます。

  • 「アクション」 - タイマーのインターバルが経過した際の動作を設定します。

  • 「移動」 - ラベルにジャンプします。

  • 「呼び出し」 - 共通イベントを呼び出します。

 

「タイマーを一時停止」

タイマーをすぐに一時停止します。

  • 「数」 - 一時停止するタイマーのインデックス（ID）です。

 

「タイマーを再開」

タイマーをすぐに再開します。

  • 「数」 - 再開するタイマーのインデックス（ID）です。

 

「タイマーを停止」

タイマーをすぐに停止します。

  • 「数」 - 終了するタイマーのインデックス（ID）です。

 

「待機」

シーンを止める、共通イベントの処理を停止するなど、挿入する位置によってさまざまな用途として使用できます。このコマンドは、とあるシーンの特定のコマンドを起動したくない場合に便利です。

詳細については、「ヒントとコツ」のページを参照してください。

 

「スクリプト」

ユーザーがコマンド内に入力したコマンドラインを実行します。このコマンドは上級者向けです。

 

「入力を待機」

キー／ボタンが押される／離されるまで待機します。

  • 「キー」 - キー／ボタンを設定します。

  • 「が」 - 設定したキー／ボタンのステータスを設定します。

  • 「押されている」 - 設定したキー／ボタンが押されるまで待機します。

  • 「離されている」 - 設定したキー／ボタンが離されるまで待機します。

 

「変数をリセット」

「ローカル変数」「すべてのローカル変数」「グローバル変数」「永続変数」のいずれかをリセットします。カスタムしたイベント／シーンベースのインゲームUIを使用している状態で、（ユーザーが新しくゲームを開始する際など）すべての変数をリセットしたい場合に便利です。

  • 「範囲」 - リセットする変数の範囲を設定します。範囲内の変数のみが影響を受けます。

  • 「ローカル」 - 指定したシーンのローカル変数のみがリセットされます。

  • 「すべてのローカル」 - あらゆるシーン／共通イベントのすべてのローカル変数がリセットされます。

  • 「グローバル」 - あらゆるグローバル変数がリセットされます。

  • 「永続」 - あらゆる永続変数がリセットされます（「永続データを保存」コマンドを呼び出す必要はありません）。

  • 「シーン」 - ローカル変数をリセットするシーンを指定します。これは「範囲」で「ローカル」が選択されている場合のみ表示されます。

  • 「種類」 - リセットする変数の種類です。

  • 「すべて」 - すべての種類の変数をリセットします。

  • 「スイッチ」 - すべてのスイッチをリセットします。

  • 「数」 - すべての数値をリセットします。

  • 「テキスト」 - すべてのテキストをリセットします。

  • 「リスト」 - すべてのリストをリセットします。

  • 「ターゲット」 - リセット対象を、指定した範囲の変数IDに限定するかどうかを設定します。

  • 「すべて」 - すべての変数が影響を受けます。

  • 「範囲」 - 指定した変数範囲のみが影響を受けます。
