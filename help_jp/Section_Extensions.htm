セクションエクステンション

 

カードエクステンションの例では、まったく新しいデータベースのカテゴリーを追加しました。しかし、たとえば新しい選択肢を「システム」に追加する時など、

既存のデータベースカテゴリーを修正するのみにとどめるほうが理にかなっている場合があります。そうした場合は「システム」カテゴリーを編集すればよいだけですが、

他のエクステンションとのコリジョンが起こる可能性があります。こうしたコリジョンを避けるために、セクションエクステンションを作成することができます。

 

セクションエクステンションは、既存のUIビューに新しいセクション（複数可）を追加することのみを目的としたエクステンションです。セクションエクステンションを作成するには、

セクションエクステンションによってインストールしたいすべてのセクションに対し、UIDプロパティを使って個別の識別子を追加する必要があります。

「システム」データベースカテゴリーに入れたセクションの例として、次を参照してください。

 

{

    "name": "Custom Section",

    "uid": "ext.CustomSystemSection",

    "items": [

        {

            "type": "GSLabel",

            "text": "Custom Field:",

            "group": "labels",

            "frame": [0, 0]

            

        },

        {

            "type": "GSTextField",

            "frame": [1, 0],

            "attribute": "ext.customValue",

        }

    ]

}

 

「システム」カテゴリーに当該のセクションのみを追加するセクションエクステンションを作成するには、エクステンションコンポーザーを開き、

「システム」のデータレコードビューをエクステンションに追加します。次に、「あなたのエクステンション」ツリービューの「システム」データレコードビューで右クリックします。

「プロパティ」を選択すると、次のようなポップアップウィンドウが表示されます。

 

「セクションのみインストール」にチェックを入れ、「<New>」をダブルクリックします。その後、ドロップダウン項目から

「Custom Section」を選択し、「OK」をクリックします。

もう一度「OK」をクリックすると完了です。

 

これでエクステンションをインストールすれば、選択したセクションのみが「システム」カテゴリーにインストールされます。「セクションのみインストール」にチェックを入れない場合、

「システム」カテゴリー全体が差し替えられます。これは既定の動作ですが、この例では必要ありません。

 

ご覧のように、セクションエクステンションにより、カスタムセクションを同じUIビューに追加することで、複数の独立したエクステンションで同じUIビューに修正を加えることが可能になります。

セクションプラグインを開発している場合、より適切にコントロールするために独自の属性名をつけるとよいでしょう。そうしなければ、同じ項目に2つ以上のエクステンションがデータを書き込んでいる場合、

属性名のコリジョンが起こることがあります。
