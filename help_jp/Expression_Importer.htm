表情インポーター

 

表情インポーターを使うと、キャラクターの画像をまとめてインポートしてデータベースエントリーを作成できます。表情インポーターでは、リソース名のパターンを指定して表情の作成方法や保存先のサブフォルダを管理できます。この機能は、表情データベースエントリーの作成に使用できる画像が200枚以上ある場合に便利です。

 

インポーターにアクセスするには、次の順に選択します。
「ツール」 → 「スクリプトエージェント」 → 「Default」 →　「表情インポーター」

 


  • 「待機ファイルパターン」 - キャラクター表情データベース内にある画像を待機アニメーション用に変換するための、画像名のパターンを設定します。

  • 「待機アニメーションパターン」 - 待機アニメーションのパターンを定義します（例：「0-1-2-1-0」）。ここで指定された順番でアニメーションフレームが作成されます。

  • 「トリガー時間」 - 待機表情のトリガー時間を設定します。ここでミリ秒単位で指定した時間の範囲内で待機表情がランダムに再生されます。たとえば、「2000 ms - 4000 ms」に設定すると、最短 2000 msから最長 4000 msの間に待機表情が再生されます。
    「会話ファイルパターン」 - キャラクター表情データベース内にある画像を会話アニメーション用に変換するための、画像名のパターンを設定します。

  • 「速度」 - アニメーションのスピードを1%～ 1000%の間で設定します。デフォルト値は「100」です。

  • 「イメージ」 - キャラクター表情データベースにインポートするイメージを設定します。事前に リソースマネージャーでインポートされたイメージのみを選択できます。

 

仕組み

たとえばファイル名が「WillowUniform_Angry」（ウィローユニフォーム＿怒り）の場合、「{フォルダ1}{フォルダ2}_{表情}」というパターンを使用します。これにより、データベースに次のような形式で新しいエントリーが作成されます。

「Willow」（ウィロー）

-「Uniform」（ユニフォーム）

--「Angry」（怒り）

 

または、次のような名前のアニメーションファイルがあったとします。

「WillowUniform_Angry_Idle_1」（ウィローユニフォーム_怒り_待機_1）

「WillowUniform_Angry_Idle_2」（ウィローユニフォーム_怒り_待機_2）

「WillowUniform_Angry_Idle_3」（ウィローユニフォーム_怒り_待機_3）

 

この場合は、「{フォルダ1}{フォルダ2}_{表情}_待機_{フレーム番号}」というパターンを使用します。これにより次のようなエントリーが自動的に作成されます。

 

「Willow」（ウィロー）

-「Uniform」（ユニフォーム）

--「Angry」（怒り） → 3ファイルの待機フレームがすべて含まれます。

 

アニメーションパターン（例：「0-1-2-1-0」）を設定して指定の順番で待機フレームを自動的に作成することもできます。

「待機ファイルパターン」ボックスと
「会話ファイルパターン」 ボックス（該当する場合）にチェックが入っていることを確認してください。
チェックが入っていないとインポーターは機能しません。

アニメーションフレームのない表情の場合：パターンから「_{フレーム番号}」 を削除します。

 

使用方法

ここでは簡単な例としてデフォルトの設定を使用します。

 

  • 「{フォルダ1}{フォルダ2}_{表情}_{フレーム番号}」のパターンに従って画像のファイル名を設定します。画像は事前に リソースマネージャーでインポートされている必要があります。

  • インポーターにアクセスするには、次の順に選択します。
    「ツール」  → 「スクリプトエージェント」 → 「Default」 → 「表情インポーター」

  • 「待機ファイルパターン」にチェックが入っていない場合はチェックを入れます。

  • 「イメージ」ボックスをクリックし、キャラクターの表情のデータベースに設定する画像を選択します。
    Shiftキー（）を押したまま選択すると複数の画像を選択できます。

  • ウィンドウが次の状態になっていることを確認し、「OK」をクリックします。

  • 次の画像が表示されたら「OK」をクリックします。
  • データベースを開き、 「キャラクターの表情」を確認します。キャラクターの表情がインポートされています。
