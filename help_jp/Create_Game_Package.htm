ゲームパッケージを作成

 

完成したゲームは、世界中にいるユーザーとシェアすることができます。

 

 

「全般」

  • 「ゲームタイトル」 - ゲームのタイトルです。

  • 「出力フォルダー」 - パッケージが保存されるフォルダーです。

  • 「プラットフォーム」 - ゲームをエクスポートするターゲットプラットフォームです。

  • 「アーキテクチャ」 -  指定のコンピュータアーキテクチャでゲームパッケージを作成します。この設定はWebプラットフォームやモバイルプラットフォームでは無視されます。

  • 「自動全画面」 - チェックを入れると、ゲームはフルスクリーンで開始されます。この設定はWebプラットフォームやモバイルプラットフォームでは無視されます。.

  • 「コンテンツを暗号化」 - チェックを入れると、ゲームのコンテンツが暗号化されます。

 

ゲームパッケージの構造

 

作成されたゲームパッケージは指定の場所に配置されます。ゲームパッケージのファイル構造は、選択したプラットフォームによって異なります。パッケージ構造をコントロールするには「ビルドと起動」セクションにチェックを入れます。次の例は、デフォルトのWIndowsパッケージのファイル構造です。

 

 

ゲームはGame.exeファイルを実行して開始できます。.dat、.dll、.bin、.pakについては詳細を把握する必要はありません。これらは、WindowsでラノゲツクールMVをプレイするのに使用されるランタイム（NW.js）の一部です。ここで重要なのは、自身が作ったゲームを周りに広める際には、これらすべてをまとめて提供しなければならない、ということです。

 

以下に他のファイルの概要を示します。

 

「Game.exe」

このファイルを実行するとゲームをプレイできます。ただし、このファイルはゲームデータを保存せず、ランタイムを実行するだけです。ゲームを提供する際はフォルダー内のすべてのファイルをまとめて提供する必要があります。

 

「index.html」

ゲームの開始ポイントです。ラノゲツクールMVはサポート対象のすべてのプラットフォームでHTML5技術を使用するため、HTMLファイルになっています。

 

「package.json」

固有のファイル、Windows、Linux、またはMac OS Xのパッケージです。このファイルは通常は操作する必要がなく、NW.jsランタイムに特殊な設定を行う場合などに使用します。

 

「icon.png」

ゲームウィンドウに使用されるアイコンです。Game.exe/Game.app.のアイコンには影響しません。これらのアイコンを編集するには、Windowsの場合、Resource Hackerなどのツールを使用してアイコンを希望のものと置き換えます。

 

「signature」

ゲームを公開する場合に重要なファイルです。ラノゲツクールMVでは、すべてのゲームにデジタル署名が使用されます。この署名は、ゲームがラノゲツクールMVの適法なバージョンで作成されていること、さらにゲームが改変またはハッキングされていないことを証明するためのものです。

 

「data」

ゲームのすべてのデータを含みます。

 

「resources」

ゲームのすべてのリソースを含みます。
